# SoftTurn Frontend\n\nNext.js 14 TypeScript frontend application for the SoftTurn electronic queue system.\n\n## Features\n\n- **Authentication**: JWT-based login with refresh token rotation\n- **Dashboard**: Real-time KPI metrics and analytics\n- **CRM Interface**: Manage tickets, workplaces, services, users\n- **Role-Based Access Control (RBAC)**: Operator, Admin, Sysadmin roles\n- **Territory-Based Access (ABAC)**: Country → City → District → Branch hierarchy\n- **Organization Management**: Full CRUD for organizational structure\n- **Ticket Management**: Create, manage, track, redirect, cancel tickets\n- **Notifications**: SMS/Viber template management and testing\n- **Audit Logs**: Comprehensive action logging\n- **Analytics**: KPI reporting and data exports\n- **Responsive Design**: Mobile-friendly interface\n\n## Tech Stack\n\n- **Next.js 14**: React framework with SSR\n- **TypeScript**: Type-safe development\n- **Tailwind CSS**: Utility-first CSS\n- **Zustand**: State management\n- **React Query**: Data fetching and caching\n- **Axios**: HTTP client with interceptors\n- **date-fns**: Date utilities\n\n## Project Structure\n\n```\nsrc/\n├── pages/           # Next.js pages\n│   ├── _app.tsx    # App entry point\n│   ├── index.tsx   # Home redirect\n│   ├── login.tsx   # Login page\n│   ├── dashboard.tsx\n│   ├── tickets.tsx\n│   ├── users.tsx\n│   └── ...\n├── components/      # Reusable React components\n│   ├── DashboardLayout.tsx\n│   └── ...\n├── services/        # API client and services\n│   └── api.ts      # Axios-based API client\n├── store/          # Zustand stores\n│   └── authStore.ts\n├── hooks/          # Custom React hooks\n│   └── index.ts\n├── utils/          # Utility functions\n│   └── formatters.ts\n├── types/          # TypeScript type definitions\n│   └── index.ts\n└── styles/         # CSS files\n    └── globals.css\n```\n\n## Setup\n\n### Prerequisites\n\n- Node.js 18+\n- npm or yarn\n- Backend running on `http://localhost:3000/api/v1` (configurable via `.env.local`)\n\n### Installation\n\n```bash\n# Install dependencies\nnpm install\n\n# Copy environment variables\ncp .env.example .env.local\n\n# Edit .env.local with your configuration\n```\n\n### Environment Variables\n\n```env\nNEXT_PUBLIC_API_BASE_URL=http://localhost:3000/api/v1\nNEXT_PUBLIC_APP_NAME=SoftTurn\nNEXT_PUBLIC_SUPPORT_EMAIL=support@softturn.ua\n```\n\n### Development\n\n```bash\n# Start development server\nnpm run dev\n\n# Server will be available at http://localhost:3001\n```\n\n### Production Build\n\n```bash\n# Build application\nnpm run build\n\n# Start production server\nnpm run start\n```\n\n### Linting\n\n```bash\n# Type checking\nnpm run type-check\n\n# Linting\nnpm run lint\n```\n\n## API Integration\n\nAll API calls are handled through the `ApiClient` class in `src/services/api.ts`. The client automatically:\n\n- Adds `Authorization` header with Bearer token\n- Handles token refresh on 401 responses\n- Persists tokens to localStorage\n- Intercepts and formats errors\n\n### Authentication Flow\n\n1. User submits login form\n2. Backend returns `accessToken` and `refreshToken`\n3. Tokens stored in localStorage and Zustand store\n4. `accessToken` sent with all subsequent requests\n5. On expiry (401), `refreshToken` used to get new `accessToken`\n6. If both tokens invalid, user redirected to login\n\n## State Management\n\n### Auth Store (Zustand)\n\n```typescript\nconst { user, login, logout, hasRole, accessToken } = useAuthStore();\n```\n\nPersisted to localStorage for session recovery.\n\n## Pages to Implement\n\n- [ ] **Tickets** (`/tickets`) -
